A reference for what goes into a website — and what AI tends to leave out.

Experienced designers know what makes a website feel real — the favicon, the 404 page, the empty states, the hover transitions. People building with AI for the first time often don't, because nobody told them those ingredients exist. AI generates the obvious layer competently and skips the rest.

This is a reference for everything that should be on the table when you're building, organized like a recipe so you can use what you need.

About this site.
It was built with AI, by a non-coder, as a class project on managing AI tools. The ingredients listed here are the same ones that had to be deliberately requested to make this site feel finished. Without the list, AI makes a website. With the list, AI makes a website you'd actually ship.

Treat this like a cookbook, not a checklist. Not every site needs every ingredient — a personal project doesn't need analytics, a static page doesn't need empty states.

The point is that you should be choosing what to skip, not skipping by accident. Expertise is knowing what's on the table. This is the table.
